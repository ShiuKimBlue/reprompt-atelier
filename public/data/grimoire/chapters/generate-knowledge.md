# 生成知识提示（Generated Knowledge Prompting）

## 核心概念

LLM 持续改进，一种流行技术是"能够融合知识或信息，以帮助模型做出更准确的预测"。基于类似思路，模型能否在预测前先生成相关知识？这正是 Liu 等人 2022 论文所探索的方向——将生成的知识作为提示的一部分，尤其对常识推理任务的效果。

![生成知识提示框架](/images/grimoire/gen-knowledge.png)

## 示例演示

### 初始问题（直接提示）

**提示：**

```
高尔夫球的一部分是试图获得比其他人更高的得分。是或否？
```

**输出：**

```
是。
```

该错误反映了 LLM 在需要世界知识的任务上的局限性。

### 第一步：生成知识

提示通过 few-shot 示例引导模型为最后一个输入生成知识：

```
输入：希腊和墨西哥哪个国家面积更大？
知识：希腊约131,957平方公里，墨西哥约1,964,375平方公里，墨西哥比希腊大1,389%。

输入：为什么眼镜在温差变化时会起雾？
知识：水蒸气在冷表面凝结形成小液滴。

输入：鱼有思考能力吗？
知识：鱼的认知能力可与非人类灵长类动物相比，拥有长期记忆。

输入：吸烟与肺癌的关系？
知识：每天不到一支烟的人患肺癌风险是不吸烟者的9倍；每天1-10支风险约为12倍。

输入：石头和卵石有什么区别？
知识：卵石根据Udden-Wentworth尺度为4-64毫米，与颗粒（2-4毫米）和巨砾（64-256毫米）不同。
```

针对高尔夫球问题，模型生成了两条知识：

**知识1：**

```
高尔夫目标是以最少杆数完成一组洞（通常18洞），总杆数决定获胜者。
```

**知识2：**

```
高尔夫是精密球杆运动，以最低分数完成比赛，将每洞所用杆数相加计算，得分最低者获胜。
```

### 第二步：整合知识得出预测

将问题重新格式化为QA格式，分别结合两条知识：

**答案1（置信度非常高）：**

```
不是，目标不是获得更高得分，而是以最少杆数完成一组洞，总杆数决定获胜者而非总得分。
```

**答案2（置信度要低得多）：**

```
是。（但随后又提到总杆数决定获胜者且得分最低者赢，内容自相矛盾。）
```

## 关键观察

两条生成的知识导致了不同的置信度结果。第一个答案置信度极高，第二个答案置信度明显较低。作者简化了演示流程，实际应用中在得出最终答案时还有更多细节需要考虑。
