# 链式思考（CoT）提示

Wei等人（2022）引入的CoT提示通过中间推理步骤实现复杂推理能力。该技术可与少样本提示结合使用，适用于需要推理的复杂任务。

![思维链提示示例](/images/grimoire/cot.png)

## 示例

**提示：**

```
这组数中的奇数加起来是偶数：4、8、9、15、12、2、1。
A：将所有奇数相加（9、15、1）得到25。答案为False。

这组数中的奇数加起来是偶数：15、32、5、13、82、7、1。
A：
```

**输出：**

```
将所有奇数相加（15、5、13、7、1）得到41。答案为False。
```

提供推理步骤后可以获得完美结果。实际上仅一个示例就足以完成此任务。

## 零样本 COT 提示

![零样本CoT提示](/images/grimoire/zero-cot.png)

Kojima等人（2022）提出的方法——在原始提示中添加"让我们逐步思考"即可触发链式推理，无需准备示例。

**示例**

直接提问：

```
我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。
然后我去买了5个苹果并吃了1个。我还剩下多少苹果？
```

输出：`11个苹果`（不正确）

添加零样本COT：

```
我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。
然后我去买了5个苹果并吃了1个。我还剩下多少苹果？

让我们逐步思考。
```

输出：

```
首先，您从10个苹果开始。
您给了邻居和修理工各2个苹果，所以您还剩下6个苹果。
然后您买了5个苹果，所以现在您有11个苹果。
最后，您吃了1个苹果，所以您还剩下10个苹果。
```

该方法在缺少可用示例时特别有用。

## 自动思维链（Auto-CoT）

![自动CoT流程](/images/grimoire/auto-cot.png)

手工制作CoT示例繁琐且可能产生次优结果。Auto-CoT利用LLM自动生成推理链，通过多样性采样来减轻错误影响。

两个阶段：

1. **问题聚类**：将问题划分为若干聚类
2. **演示抽样**：从每组中选代表性问题，用Zero-Shot-CoT配合简单启发式生成推理链
