# 大语言模型的检索增强生成 (RAG) 方法

## RAG 简介

在大语言模型（LLMs）的实际应用中，面临诸多挑战，涵盖领域知识不足、信息准确性问题以及虚假内容生成等。RAG 通过引入外部知识库等额外信息源，为这些问题提供了有效的缓解策略。该方法在需要持续更新知识的知识密集型场景或特定领域中尤为有效。RAG 的一大优势在于无需针对特定任务重新训练大语言模型。

RAG 将输入与一组相关的支持文档结合，这些文档通常来自维基百科等来源。文档被添加到输入提示中，一同送入文本生成器产生最终输出。这一机制特别适用于信息不断更新的场景，因为 LLM 所依赖的参数知识本质上是静态的。通过 RAG，语言模型无需重新训练即可访问最新信息，生成可靠的基于检索的输出。

简言之，RAG 通过检索到的证据提升 LLM 响应的准确性、可控性和相关性，在快速变化的环境中解决问题时尤其有价值，能有效减少错误信息生成和性能下降。

**RAG 应用的典型工作流程包括以下步骤：**

- **输入：** LLM 系统需要回答的问题。如果不使用 RAG，问题直接由 LLM 回答
- **索引：** 使用 RAG 时，先将相关文档分块，为这些块生成嵌入向量，并将它们索引到向量库中。查询时，查询内容也以相似方式嵌入
- **检索：** 通过比较查询内容与索引向量，找到相关文档
- **生成：** 将相关文档与原始提示结合作为额外上下文，传递给模型进行回应生成

## RAG 范式的演变

近几年来，RAG 系统经历了从初级阶段到高级阶段，再到模块化阶段的演变。

### 初级 RAG

初级 RAG 采用传统过程，包括索引建立、文档检索和内容生成。其局限性包括低精确度和低召回率。

### 高级 RAG

高级 RAG 解决了初级 RAG 的问题，尤其在提高检索质量方面，包括优化检索前、检索时和检索后的过程。

- **检索前准备阶段** — 通过优化数据索引提高数据质量
- **检索阶段** — 通过改进嵌入模型提高上下文片段质量
- **检索后优化过程** — 关注上下文窗口限制和减少噪音

### 模块化的 RAG

模块化 RAG 通过增强功能模块来提升性能，支持混合搜索、递归检索、StepBack-prompt、子查询策略、假设性文档嵌入技术 (HyDE) 等。

## RAG 框架简述

### 检索技术

- **数据分块策略** — 确定合适的数据分块方式取决于数据内容和应用需求
- **专业领域嵌入模型微调** — 若工作聚焦于特定领域，需对嵌入模型进行微调
- **查询重写** — 通过 Query2Doc、ITER-RETGEN 和 HyDE 等工具改写查询
- **检索器与 LLM 协同优化** — 通过 AAR、REPLUG 和 UPRISE 等方式

### 文本生成

- **检索后处理与模型固定** — 保持 LLM 不变，通过信息简化和结果优先排序改善检索结果
- **针对 RAG 系统的 LLM 微调** — 确保生成文本既自然流畅又能有效结合检索到的文档信息

### 增强技术

- **迭代检索** — 模型通过多轮检索不断深化和丰富信息内容
- **递归检索** — 一次检索的输出成为另一次检索的输入
- **自适应检索** — 根据特定需求调整检索过程

### RAG 与微调的区别

RAG 特别适合融合新知识，微调则能优化模型内部知识、输出格式及提升复杂指令执行能力。两种方法可以相辅相成。

## RAG 模型评估解析

三个质量指标：上下文相关性、答案忠实度、答案相关性。

四项能力：对噪声的鲁棒性、负面信息的排除、信息整合能力、面对假设情况的鲁棒性。

使用 RGB、RECALL 等基准测试，以及 RAGAS、ARES、TruLens 等评估工具。

## RAG 研究的挑战与前景

- 上下文长度问题、系统鲁棒性、混合方法探索、生产级挑战、多模态 RAG 发展、评价机制完善
