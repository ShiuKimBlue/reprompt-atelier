# Meta Prompting（元提示）

## 概述

Meta Prompting 是一种高级提示技术，强调"任务和问题的结构与语法方面，而非其具体内容细节"。其目标是创建一种更抽象、结构化的方法来与 LLM 交互，优先考虑形式和模式，而非以内容为中心的方法。

## 关键特征

根据 [Zhang 等人（2024）](https://arxiv.org/abs/2311.11482)的研究，Meta Prompting 由五个关键特征定义：

1. **结构导向**——优先考虑问题和解决方案的格式与模式，而非具体内容。
2. **语法聚焦**——使用语法作为期望响应的指导模板。
3. **抽象示例**——使用抽象化的示例作为框架，展示问题/解决方案的结构，而非具体细节。
4. **通用性强**——适用于各种领域能够为不同问题提供结构化响应。
5. **分类方法**——借鉴类型论，强调提示组件的分类和逻辑排列。

## 相比少样本提示的优势

[Zhang 等人（2024）](https://arxiv.org/abs/2311.11482)强调 Meta Prompting 专注于"结构导向的方法，而非少样本提示中使用的内容驱动方法"。

主要优势包括：

1. **Token 效率**——通过关注结构而非详细内容来减少 token 消耗。
2. **公平比较**——通过最小化特定示例的影响，实现更公平的问题解决模型比较。
3. **零样本效果**——作为一种零样本提示形式运作，最小化特定示例的影响。

## 应用场景

Meta Prompting 通过利用问题解决的结构模式来增强 LLM 在各领域的推理能力。它假设 LLM 拥有关于任务的先天知识。对于更新颖的任务，性能可能会下降，类似于零样本提示的局限性。

有益的应用领域包括：

- 复杂推理任务
- 数学问题解决
- 编程挑战
- 理论查询
