# Kimi K2.5

## 概述

Moonshot AI 发布了 Kimi K2.5，一个开源多模态代理模型，旨在推进通用代理智能。它构建在 Kimi K2 之上——一个万亿参数的 MoE Transformer，在 15 万亿 token 上预训练——K2.5 联合优化文本和视觉，使两种模态相互增强而非争夺容量。

该模型引入了 Agent Swarm，一个自主并行代理编排框架，能够动态将复杂任务分解为异构子问题并并发执行，相比单代理基线将延迟降低多达 4.5 倍。

经过后训练的 Kimi K2.5 模型检查点已在 Hugging Face 上公开发布，供未来研究和实际应用使用。

## 主要贡献

Kimi K2.5 通过两个主要贡献推进了最新技术：文本和视觉的联合优化，以及用于并行代理编排的 Agent Swarm。这些共同使模型在推理、编码、多模态理解、代理任务和计算机使用方面表现出色。

## 文本和视觉的联合优化

大多数视觉适配模型将多模态能力视为附加组件，在训练后期以高比例（例如 50% 或更高）引入视觉 token。Kimi K2.5 采用了不同的方法。在固定的总视觉-文本 token 预算下，早期融合与较低的视觉比例产生更好的结果。与其在最后进行激进的视觉密集训练，K2.5 在整个训练过程中以恒定的适中比例混合文本和视觉 token，让模型在延长的共同优化期间自然发展平衡的多模态表示。

在架构上，Kimi K2.5 采用了 **MoonViT-3D**，一个三维原生分辨率视觉编码器，以原始分辨率处理图像，无需复杂的子图像分割。对于视频理解，最多四个连续帧被视为时空体积，2D patch 被展平并打包成单个 1D 序列。图像和视频编码器之间的这种共享，结合 4 倍时间压缩，使 K2.5 能够在相同的上下文窗口内处理长达 4 倍的视频。

### Zero-Vision SFT

一个关键发现：纯文本 SFT 数据足以激活视觉代理能力，消除了后训练中对人工标注视觉思维链数据的需求。所有图像操作都通过 IPython 中的编程操作代理，这作为传统视觉工具使用的泛化。这支持多样化的推理行为，包括像素级操作、物体定位、计数和 OCR——全部从纯文本监督激活。

### 视觉 RL 提升文本性能

一个反直觉的发现：基于结果的视觉强化学习实际上提升了纯文本基准测试的性能。视觉 RL 后，纯文本任务的改进包括：

- MMLU-Pro: 84.7% → 86.4%
- GPQA-Diamond: 84.3% → 86.4%
- LongBench v2: 56.7% → 58.9%

这表明视觉 RL 增强了结构化信息提取中的校准，有助于跨模态泛化，在不降低语言能力的情况下提升文本推理。

## Agent Swarm

大多数现有的代理模型依赖于推理和工具调用步骤的顺序执行。即使具有数百个推理步骤的系统也会遭受推理时间的线性扩展。Kimi K2.5 引入了 Agent Swarm，一个动态的并行代理编排框架，区别于顺序链和预指定的并行化启发式方法。

### Agent Swarm 的工作原理

K2.5 不是将任务作为推理链执行，而是通过动态任务分解、子代理实例化和并行子任务调度来启动 Agent Swarm。一个可训练的编排器创建专门的冻结子代理并分配任务。编排器决定什么要并行化以及何时并行化——这些决策通过环境反馈和 RL 驱动的探索学习，而非硬编码。

该架构有意将可训练的编排器与从固定中间策略检查点实例化的冻结子代理解耦。在训练期间，子代理被冻结，其执行轨迹被视为环境观察而非可微分的决策点。只有编排器通过 RL 更新。这规避了信用分配模糊性和训练不稳定性（多代理设置中的噪声、稀疏奖励）。

### PARL：并行代理强化学习

训练编排器有效并行化并非易事。PARL 奖励函数结合三个组件：

1. **并行性奖励** — 激励生成并发子代理（防止默认为顺序执行的"序列坍缩"）
2. **完成奖励** — 确保子代理成功完成分配的子任务（防止"虚假并行"）
3. **性能奖励** — 评估解决方案的整体质量

训练使用关键步骤（类似于计算图中的关键路径分析）而非总步骤作为成本指标。这激励了平衡良好的任务分解，缩短最长的并行分支，最小化端到端延迟而非仅仅最大化并发度。

### Agent Swarm 结果

- **BrowseComp:** 78.4% — 比单代理 K2.5 基线（60.6%）绝对提升 17.8%，超越 GPT-5.2 Pro（77.9%）
- **WideSearch (Item-F1):** 72.7% → 79.0%，超越 Claude Opus 4.5（76.2%），建立新的 SOTA
- 与单代理基线相比，执行时间减少 3-4.5 倍，同时提高了准确性

Agent Swarm 还充当主动上下文管理。长时间任务被分解为并行、语义隔离的子任务，每个由具有有界本地上下文的专门子代理执行。只有与任务相关的输出被路由回编排器。

## Token 高效 RL 与 Toggle

Kimi K2.5 引入了 **Toggle**，一种训练启发式方法，在推理时缩放和预算约束优化之间交替。在预算有限阶段，模型在任务相关的 token 预算内解决问题，产生更简洁的思维链推理。在标准缩放阶段，它为更难的问题生成最多到最大 token 限制的响应。

Toggle 将输出 token 减少 25-30%，性能影响可忽略不计，即使仅在数学和编程任务上训练也显示出强大的领域泛化能力。

## 模型架构概述

Kimi K2.5 构建在 Kimi K2 语言模型之上——一个 **1.04 万亿总参数的 MoE 模型**，使用 384 个专家，每个 token 激活 8 个（320 亿激活参数）。

多模态架构由三个组件组成：

1. **MoonViT-3D**（视觉编码器）
2. **MLP 投影器**
3. **Kimi K2 MoE 语言模型**

预训练在三个阶段处理约 15 万亿 token：

1. **独立 ViT 训练** — 1T token
2. **联合预训练**，4K 序列长度 — 15T token
3. **中期训练**，使用高质量数据，长上下文激活至 262K 序列长度
