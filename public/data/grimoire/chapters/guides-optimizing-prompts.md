# 优化提示词

**为 LLM 打造有效提示词**

大语言模型在许多任务上提供了强大的能力，但其效果取决于提示词的质量。本指南涵盖设计提示词以最大化 LLM 性能的关键方面。

---

## 提示词设计的关键考量

**具体性和清晰性：** 提示词应"清晰地表达期望的结果"——歧义可能产生意外或不相关的结果。

**结构化输入和输出：** 使用 JSON 或 XML 等格式有助于 LLM 理解和处理信息。指定期望的输出格式（列表、段落、代码片段）也能提高相关性。

**使用分隔符增强结构：** 在提示词中使用特殊字符作为分隔符"进一步澄清结构并分隔不同元素"，帮助模型理解。

**复杂操作的任务分解：** 与其呈现一个单一的大型提示词，"将复杂流程分解为更简单的子任务可显著提高清晰度和性能"。

---

## 高级提示技术

**少样本提示（Few-Shot Prompting）：** 提供期望输入-输出对的示例，通过展示期望的模式引导模型产生更高质量的响应。[了解更多关于少样本提示](/grimoire/few-shot)。

**思维链提示（Chain-of-Thought Prompting）：** 提示模型"逐步思考"，通过将复杂任务分解为中间推理步骤来增强逻辑推理能力。[了解更多关于思维链提示](/grimoire/chain-of-thought)。

**ReAct（推理 + 行动）：** 此方法从 LLM 中引出"高级推理、规划甚至工具使用"。构建鼓励这些能力的提示词可解锁更复杂的应用。[了解更多关于 ReAct](/grimoire/react)。

---

## 结论

有效的提示词设计对于充分利用 LLM 的潜力至关重要。通过遵循最佳实践——具体性、结构化格式、任务分解——并利用少样本、思维链和 ReAct 提示等高级技术，"开发者可以显著提高输出的质量、准确性和复杂性"。
