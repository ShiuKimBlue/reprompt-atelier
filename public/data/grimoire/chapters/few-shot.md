# 少样本提示

虽然大型语言模型展示了惊人的零样本能力，但在更复杂的任务上仍然表现不佳。少样本提示通过在提示中提供演示示例来引导模型实现更好的性能，即所谓的上下文学习（in-context learning）。

## 示例：学习新词

以下演示如何通过提供示例让模型学会在句子中使用新词（1-shot）：

```
"whatpu"是坦桑尼亚的一种小型毛茸茸的动物。一个使用whatpu这个词的句子的例子是：
我们在非洲旅行时看到了这些非常可爱的whatpus。
"farduddle"是指快速跳上跳下。一个使用farduddle这个词的句子的例子是：
```

**输出：**
```
当我们赢得比赛时，我们都开始庆祝跳跃。
```

对于更复杂的任务，可以尝试增加演示数量（如3-shot、5-shot、10-shot等）。

## 演示设计的关键发现

根据 Min 等人（2022）的研究：

- 标签空间和演示指定的输入文本的分布都很重要
- 格式对性能起着关键作用，即使使用随机标签也比没有标签好
- 从真实标签分布中选择随机标签也有帮助

## 少样本提示的限制

标准少样本提示在处理更复杂的推理任务时仍有不足。这类任务涉及多个推理步骤，少样本提示不足以获得可靠结果。建议考虑使用**链式思考（CoT）提示**等更高级的技术。
